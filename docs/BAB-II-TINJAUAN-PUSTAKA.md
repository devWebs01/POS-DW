## 2.1 Konsep Dasar Sistem Informasi

> **⏳ Template — Konten perlu diisi dengan teori dasar sistem informasi dari referensi pustaka.**

Sistem informasi merupakan kombinasi dari teknologi informasi, prosedur, dan sumber daya manusia yang bekerja bersama untuk mengumpulkan, memproses, menyimpan, dan mendistribusikan informasi guna mendukung pengambilan keputusan, koordinasi, dan pengendalian dalam suatu organisasi (Laudon & Laudon, 2020).

Komponen sistem informasi terdiri dari:
1. **Hardware** — Perangkat keras fisik yang digunakan untuk memproses input dan output.
2. **Software** — Program dan instruksi yang mengatur kerja hardware.
3. **Data** — Fakta mentah yang diolah menjadi informasi.
4. **Prosedur** — Aturan dan kebijakan yang mengatur operasi sistem.
5. **Manusia** — Pengguna yang mengoperasikan dan memanfaatkan sistem.
6. **Jaringan** — Infrastruktur komunikasi yang menghubungkan komponen sistem.

## 2.2 Point of Sale (POS)

> **⏳ Template — Konten perlu diisi dengan teori POS dari referensi pustaka.**

Point of Sale (POS) adalah sistem yang digunakan untuk mencatat dan memproses transaksi penjualan di lokasi penjualan (retail). Sistem POS modern tidak hanya berfungsi sebagai cash register digital, tetapi juga mencakup manajemen inventori, pelaporan penjualan, manajemen pelanggan, dan integrasi dengan sistem pembayaran.

Fungsi utama sistem POS meliputi:
1. **Pencatatan Transaksi** — Mencatat item yang dibeli, jumlah, harga, dan total pembayaran.
2. **Manajemen Produk** — Mengelola data produk, kategori, harga, dan stok.
3. **Manajemen Pembayaran** — Mendukung berbagai metode pembayaran (tunai, kartu, e-wallet).
4. **Pelaporan** — Menghasilkan laporan penjualan, inventori, dan analisis bisnis.
5. **Manajemen Inventori** — Melacak stok produk dan memberikan notifikasi saat stok menipis.

## 2.3 Data Warehouse

> **⏳ Template — Konten perlu diisi dengan teori Data Warehouse dari referensi pustaka.**

Data Warehouse adalah sistem penyimpanan data yang dirancang khusus untuk mendukung analisis dan pelaporan bisnis. Data Warehouse mengintegrasikan data dari berbagai sumber ke dalam satu repositori terpusat yang dioptimalkan untuk query analitis (Inmon, 2005).

Karakteristik Data Warehouse:
1. **Subject-Oriented** — Data diorganisir berdasarkan subjek bisnis (penjualan, produk, pelanggan).
2. **Integrated** — Data dari berbagai sumber diintegrasikan ke dalam format yang konsisten.
3. **Time-Variant** — Data historis disimpan untuk analisis tren dan perubahan dari waktu ke waktu.
4. **Non-Volatile** — Data yang sudah dimasukkan tidak dapat diubah atau dihapus.

Dalam konteks sistem POS DW, konsep Data Warehouse diterapkan pada penyimpanan data transaksi historis yang digunakan untuk pelaporan penjualan, analisis produk terlaris, dan identifikasi tren bisnis.

## 2.4 Framework Laravel

> **⏳ Template — Konten perlu diisi dengan teori Laravel dari referensi pustaka.**

Laravel adalah framework aplikasi web berbasis PHP yang open-source dengan arsitektur MVC (Model-View-Controller). Laravel dikenal dengan sintaks yang elegan, fitur keamanan yang lengkap, dan ekosistem yang kaya.

Fitur utama Laravel yang digunakan dalam penelitian ini:
1. **Eloquent ORM** — Active Record implementation untuk interaksi database yang intuitif.
2. **Blade Templating** — Template engine yang ringan dan powerful.
3. **Artisan CLI** — Command-line tool untuk otomatisasi tugas pengembangan.
4. **Migration** — Version control untuk skema database.
5. **Routing** — Sistem routing yang fleksibel.
6. **Middleware** — Filter HTTP request untuk keamanan dan validasi.
7. **Laravel Fortify** — Backend autentikasi yang headless dan fleksibel.

---

## 2.5 Livewire dan Flux UI

> **⏳ Template — Konten perlu diisi dengan teori Livewire dan Flux UI dari referensi pustaka.**

Livewire adalah full-stack framework untuk Laravel yang memungkinkan pembuatan antarmuka pengguna yang dinamis dan reaktif tanpa perlu menulis JavaScript secara eksplisit. Livewire bekerja dengan mengirimkan request AJAX secara otomatis ke server dan merender ulang komponen yang berubah.

Flux UI adalah library komponen UI untuk Livewire yang menyediakan komponen-komponen siap pakai seperti tombol, input form, modal, tabel, navigasi, dan lain-lain. Flux UI dibangun di atas Tailwind CSS dan Alpine.js, sehingga memudahkan pengembang dalam membangun antarmuka yang konsisten dan responsif.

---

## 2.6 Tailwind CSS

> **⏳ Template — Konten perlu diisi dengan teori Tailwind CSS dari referensi pustaka.**

Tailwind CSS adalah framework CSS utility-first yang menyediakan kelas-kelas utilitas untuk membangun desain kustom dengan cepat. Tidak seperti framework CSS tradisional (seperti Bootstrap) yang menyediakan komponen siap pakai, Tailwind CSS memberikan fleksibilitas penuh dalam mendesain antarmuka dengan menggabungkan kelas-kelas utilitas langsung di HTML.

---

## 2.7 MySQL

> **⏳ Template — Konten perlu diisi dengan teori MySQL dari referensi pustaka.**

MySQL adalah sistem manajemen basis data relasional (RDBMS) open-source yang menggunakan Structured Query Language (SQL). MySQL dikenal dengan performa tinggi, keandalan, dan kemudahan penggunaan. MySQL 8.0 digunakan sebagai database management system dalam penelitian ini.
