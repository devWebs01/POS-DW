# 4.5 IMPLEMENTASI SISTEM

Implementasi sistem merupakan tahapan penerapan hasil perancangan sistem ke dalam bentuk aplikasi yang dapat dijalankan. ### Contoh Kode Migration

Berikut adalah contoh implementasi migration untuk tabel products:

```php
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('products', function (Blueprint $table) {
            $table->id();
            $table->foreignId('category_id')->constrained()->cascadeOnDelete();
            $table->string('name', 200);
            $table->string('slug', 220)->unique();
            $table->string('sku', 50)->unique()->index();
            $table->decimal('price', 12, 2)->default(0);
            $table->integer('stock')->default(0);
            $table->text('description')->nullable();
            $table->boolean('is_active')->default(true)->index();
            $table->timestamps();

            $table->index('category_id');
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('products');
    }
};
```

(Sumber: File `database/migrations/2025_05_09_000002_create_products_table.php`)

### Eksekusi Migration

Migration dijalankan menggunakan perintah Artisan:

```bash
php artisan migrate
```

Perintah ini akan mengeksekusi seluruh migration yang belum dijalankan secara berurutan. Setiap migration dicatat dalam tabel migrations sehingga hanya migration baru yang akan dieksekusi pada eksekusi berikutnya.

### Status Migration

Setelah seluruh migration dijalankan, status dapat diverifikasi dengan perintah:

```bash
php artisan migrate:status
```

Hasil eksekusi menunjukkan seluruh 13 migration telah berhasil dijalankan (status "Ran").

### Implementasi Eloquent Models

#### Model Category

Model Category merepresentasikan entitas kategori produk dengan relasi one-to-many ke Product.

```php
<?php

namespace App\Models;

use Database\Factories\CategoryFactory;
use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\HasMany;

class Category extends Model
{
    /** @use HasFactory<CategoryFactory> */
    use HasFactory;

    protected $fillable = ['name', 'slug', 'description'];

    public function products(): HasMany
    {
        return $this->hasMany(Product::class);
    }
}
```

(Sumber: File `app/Models/Category.php`)

#### Model Product

Model Product merupakan model inti yang memiliki relasi ke Category dan TransactionItem. Model ini juga memiliki accessor untuk mengelola URL gambar produk.

```php
<?php

namespace App\Models;

use Database\Factories\ProductFactory;
use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;
use Illuminate\Database\Eloquent\Relations\HasMany;

class Product extends Model
{
    /** @use HasFactory<ProductFactory> */
    use HasFactory;

    protected $fillable = [
        'category_id', 'name', 'slug', 'sku',
        'price', 'stock', 'image', 'description', 'is_active',
        'is_unlimited_stock',
    ];

    protected function casts(): array
    {
        return [
            'is_active' => 'boolean',
            'is_unlimited_stock' => 'boolean',
            'price' => 'float',
        ];
    }

    public function getImageUrlAttribute(): string
    {
        if (! $this->image) {
            return asset('images/product-default.svg');
        }

        if (str_starts_with($this->image, 'http')) {
            return $this->image;
        }

        if (file_exists(storage_path('app/public/'.$this->image))) {
            return asset('storage/'.$this->image);
        }

        return asset('images/product-default.svg');
    }

    public function category(): BelongsTo
    {
        return $this->belongsTo(Category::class);
    }

    public function transactionItems(): HasMany
    {
        return $this->hasMany(TransactionItem::class);
    }
}
```

(Sumber: File `app/Models/Product.php`)

#### Model User

Model User mengimplementasikan autentikasi menggunakan Laravel Fortify dan role-based access control menggunakan Spatie Permission.

```php
<?php

namespace App\Models;

use Database\Factories\UserFactory;
use Illuminate\Database\Eloquent\Attributes\Fillable;
use Illuminate\Database\Eloquent\Attributes\Hidden;
use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Relations\HasMany;
use Illuminate\Foundation\Auth\User as Authenticatable;
use Illuminate\Notifications\Notifiable;
use Illuminate\Support\Str;
use Laravel\Fortify\TwoFactorAuthenticatable;

#[Fillable(['name', 'email', 'password'])]
#[Hidden(['password', 'two_factor_secret', 'two_factor_recovery_codes', 'remember_token'])]
class User extends Authenticatable
{
    /** @use HasFactory<UserFactory> */
    use HasFactory, Notifiable, \Spatie\Permission\Traits\HasRoles, TwoFactorAuthenticatable;

    protected function casts(): array
    {
        return [
            'email_verified_at' => 'datetime',
            'password' => 'hashed',
        ];
    }

    public function initials(): string
    {
        return Str::of($this->name)
            ->explode(' ')
            ->take(2)
            ->map(fn ($word) => Str::substr($word, 0, 1))
            ->implode('');
    }

    public function transactions(): HasMany
    {
        return $this->hasMany(Transaction::class);
    }
}
```

### Implementasi Livewire Component (Volt)

Sistem menggunakan Livewire Volt, yaitu single-file component yang menggabungkan logika PHP dan template Blade dalam satu file. Berikut adalah contoh implementasi halaman transaksi POS yang menunjukkan fitur real-time:

**State Management (Halaman Transaksi POS):**

Halaman transaksi POS menggunakan state management dari Livewire untuk mengelola data keranjang secara real-time:

```php
<?php

use App\Models\Category;
use App\Models\Product;
use App\Models\Transaction;
use App\Models\TransactionItem;
use Flux\Flux;
use Illuminate\Support\Str;
use Livewire\WithPagination;

use function Laravel\Folio\middleware;
use function Laravel\Folio\name;
use function Livewire\Volt\computed;
use function Livewire\Volt\state;
use function Livewire\Volt\usesPagination;

usesPagination();

name('transactions.create');
middleware('auth');
middleware('verified');

state([
    'customer' => '',
    'category_id' => (int) '',
    'productSearch' => '',
    'cart' => [],
    'payment_method' => 'cash',
    'paid_amount' => 0,
    'notes' => '',
    'showConfirmModal' => false,
]);
```

(Sumber: File `resources/views/pages/transactions/create.blade.php`)

**Computed Properties untuk Data Real-Time:**

```php
$categoryOptions = computed(function () {
    return Category::orderBy('name')->get();
});

$products = computed(function () {
    $query = Product::query()->where('is_active', true);

    if (!empty($this->category_id)) {
        $query->where('category_id', $this->category_id);
    }

    if (!empty($this->productSearch)) {
        $query->where(function ($q) {
            $q->where('name', 'like', '%' . $this->productSearch . '%')
              ->orWhere('sku', 'like', '%' . $this->productSearch . '%');
        });
    }

    return $query->orderBy('name')->paginate(12);
});

$totalAmount = computed(function () {
    return collect($this->cart)->sum('subtotal');
});
```

**Action untuk Menambahkan ke Keranjang:**

```php
$addToCart = function ($productId) {
    $product = Product::findOrFail($productId);

    if (isset($this->cart[$productId])) {
        $this->cart[$productId]['quantity']++;
    } else {
        $this->cart[$productId] = [
            'product_id' => $productId,
            'name' => $product->name,
            'price' => $product->price,
            'quantity' => 1,
            'subtotal' => $product->price,
        ];
    }

    $this->cart[$productId]['subtotal'] =
        $this->cart[$productId]['quantity'] * $this->cart[$productId]['price'];
};
```

### Autentikasi dan Keamanan

Sistem menggunakan Laravel Fortify untuk backend autentikasi yang mencakup:

1. **Login**: Menggunakan email dan password dengan validasi dan throttling
2. **Two-Factor Authentication (2FA)**: Menggunakan Google Authenticator dengan QR Code setup
3. **Email Verification**: Verifikasi alamat email untuk mengaktifkan akun
4. **Password Reset**: Fitur lupa password dengan email token
5. **Password Confirmation**: Konfirmasi password ulang untuk aksi sensitif

Untuk otorisasi, sistem menggunakan Spatie Laravel Permission dengan implementasi Role-Based Access Control (RBAC):

```php
// Contoh pengecekan permission di blade
@can('view-products')
    <a href="/products">Kelola Produk</a>
@endcan

// Contoh pengecekan role
@role('admin')
    <a href="/users">Kelola Pengguna</a>
@endrole
```

## Ringkasan Implementasi

Berdasarkan hasil implementasi yang telah dilakukan, Sistem POS DW berhasil dibangun dengan mencakup seluruh komponen yang telah dirancang sebelumnya. Ringkasan implementasi sistem adalah sebagai berikut:

1. **Database**: Sebanyak 13 tabel berhasil diimplementasikan melalui mekanisme Laravel Migration yang memungkinkan pengelolaan skema basis data secara terprogram dan terdokumentasi.
2. **Backend**: Implementasi backend menggunakan Laravel 13 dengan enam model Eloquent yang merepresentasikan entitas sistem, serta penerapan pola arsitektur MVC (Model-View-Controller).
3. **Frontend**: Implementasi frontend menggunakan Livewire 4 dengan Volt yang memungkinkan pembuatan komponen interaktif secara real-time tanpa memerlukan pemuatan ulang halaman.
4. **Autentikasi**: Sistem autentikasi dibangun menggunakan Laravel Fortify yang mencakup fitur Two-Factor Authentication (2FA), verifikasi email, dan pengaturan ulang kata sandi.
5. **Otorisasi**: Sistem otorisasi dibangun menggunakan Spatie Permission dengan penerapan Role-Based Access Control (RBAC) yang mencakup tiga peran utama, yaitu Admin, Kasir, dan Manager.
6. **Routing**: Routing sistem menggunakan Laravel Folio dengan pendekatan page-based routing yang memetakan setiap berkas Blade secara otomatis ke rute yang sesuai.
7. **UI Components**: Komponen antarmuka menggunakan Livewire Flux UI dengan Tailwind CSS 4 yang menghasilkan tampilan modern, konsisten, dan responsif.

Seluruh implementasi dilakukan dengan mengacu pada praktik terbaik (best practices) pengembangan Laravel modern dan telah siap untuk memasuki tahap pengujian sistem.
