# 4.4 PERANCANGAN ANTARMUKA (UI/UX)

Perancangan antarmuka pengguna (User Interface/UI) merupakan tahapan penting dalam pengembangan sistem yang bertujuan untuk menciptakan tampilan yang menarik, mudah digunakan, dan sesuai dengan kebutuhan pengguna. Antarmuka yang baik akan meningkatkan produktivitas pengguna dan mengurangi kesalahan dalam pengoperasian sistem.

Sistem POS DW dirancang dengan pendekatan modern menggunakan framework Livewire Flux UI yang menyediakan komponen-komponen antarmuka yang konsisten dan responsif. Seluruh halaman dirancang untuk dapat diakses melalui berbagai perangkat (desktop, tablet, dan mobile) dengan tampilan yang optimal.

---

## 4.4.1 Halaman Login dan Autentikasi

Halaman login merupakan halaman pertama yang dilihat oleh pengguna saat mengakses sistem. Halaman ini berfungsi sebagai gerbang keamanan yang memastikan hanya pengguna terdaftar yang dapat mengakses sistem.

### Desain Halaman Login

**Gambar 4.11 Rancangan Halaman Login**

![UI Login](bab-iv/images/ui-login.png)

*Sumber: Hasil Perancangan (2026)*

### Elemen Antarmuka

Halaman login terdiri dari beberapa elemen berikut:

| No | Elemen | Tipe | Deskripsi |
|----|--------|------|-----------|
| 1 | Logo Aplikasi | Gambar | Logo sistem POS DW yang ditampilkan di bagian atas form login |
| 2 | Judul Halaman | Teks | Teks "Masuk ke Akun Anda" sebagai judul halaman |
| 3 | Input Email | Text Input | Kolom input untuk memasukkan alamat email dengan validasi format email |
| 4 | Input Password | Password Input | Kolom input untuk memasukkan password dengan ikon show/hide password |
| 5 | Checkbox "Remember Me" | Checkbox | Opsi untuk menyimpan session login |
| 6 | Tombol Login | Button | Tombol "Masuk" dengan warna primer (biru) untuk submit kredensial |
| 7 | Link Lupa Password | Link | Tautan ke halaman reset password |
| 8 | Link Register | Link | Tautan ke halaman registrasi (jika registrasi terbuka) |
| 9 | Pesan Error | Alert | Area untuk menampilkan pesan error validasi |
| 10 | Loading State | Spinner | Indikator loading saat proses login berlangsung |

### Alur Penggunaan

1. Pengguna membuka URL aplikasi dan diarahkan ke halaman login
2. Pengguna memasukkan email dan password pada kolom yang tersedia
3. Pengguna dapat mencentang "Remember Me" untuk menyimpan sesi login
4. Pengguna menekan tombol "Masuk"
5. Sistem melakukan validasi dan autentikasi
6. Jika berhasil, sistem mengarahkan ke halaman dashboard
7. Jika gagal, sistem menampilkan pesan error yang informatif

Jika pengguna telah mengaktifkan fitur 2FA, setelah memasukkan kredensial yang valid, sistem akan menampilkan halaman input kode OTP:

**Elemen Halaman 2FA:**

## 4.4.2 Dashboard

Dashboard merupakan halaman utama setelah login yang menampilkan ringkasan informasi bisnis secara visual. Dashboard dirancang untuk memberikan gambaran cepat tentang kondisi terkini toko.

### Desain Dashboard

**Gambar 4.13 Rancangan Halaman Dashboard**

![UI Dashboard](bab-iv/images/ui-dashboard.png)

*Sumber: Hasil Perancangan (2026)*

### Struktur Tampilan

Dashboard terdiri dari beberapa area utama:

**1. Navigasi (Sidebar)**

Sidebar terletak di sebelah kiri layar dan berisi menu navigasi utama:

| Menu | Ikon | Deskripsi | Hak Akses |
|------|------|-----------|-----------|
| Dashboard | Home icon | Menuju halaman dashboard | Semua user |
| Kategori | Folder icon | Manajemen kategori produk | Admin |
| Produk | Package icon | Manajemen data produk | Admin |
| Transaksi | Shopping Cart icon | Transaksi POS | Kasir, Admin |
| Riwayat Transaksi | List icon | Riwayat transaksi | Semua user |
| Laporan | Chart icon | Laporan penjualan | Admin, Manager |
| Pengguna | Users icon | Manajemen user | Admin |
| Role & Permission | Shield icon | Manajemen role dan hak akses | Admin |
| Pengaturan | Settings icon | Konfigurasi sistem | Admin |

**2. Header**

Header terletak di bagian atas halaman dan berisi:
- Judul halaman aktif
- Search bar (untuk halaman tertentu)
- Notifikasi
- Avatar dan nama user
- Dropdown menu (Profil, Settings, Logout)

**3. Konten Utama**

Konten utama dashboard menampilkan card-card statistik:

**Card Statistik:**
| Card | Data yang Ditampilkan | Sumber Data |
|------|----------------------|-------------|
| Total Penjualan Hari Ini | Jumlah rupiah penjualan | Transaksi hari ini |
| Jumlah Transaksi | Jumlah transaksi hari ini | Transaksi hari ini |
| Produk Terlaris | Produk dengan penjualan tertinggi | Transaction items |
| Stok Menipis | Jumlah produk dengan stok < 10 | Produk |

**Grafik Penjualan:**
- Grafik garis penjualan 7 hari terakhir
- Sumbu X: Tanggal
- Sumbu Y: Total penjualan (rupiah)

**Tabel Transaksi Terbaru:**
- Menampilkan 5-10 transaksi terakhir
- Kolom: Invoice, Customer, Total, Tanggal, Aksi

## 4.4.3 Manajemen Kategori dan Produk

### Halaman Daftar Kategori

Halaman kategori menampilkan daftar kategori produk dalam bentuk tabel.

**Gambar 4.14 Rancangan Halaman Kategori**

![UI Categories](bab-iv/images/ui-categories.png)

*Sumber: Hasil Perancangan (2026)*

**Elemen:**
- **Tombol "Tambah Kategori"**: Tombol dengan ikon "+" untuk menambah kategori baru
- **Search Bar**: Kolom pencarian untuk mencari kategori berdasarkan nama
- **Tabel Kategori**: Menampilkan data kategori dalam tabel dengan kolom:
  - Nama kategori
  - Slug (ditampilkan sebagai tag/badge)
  - Jumlah produk (link ke halaman produk terfilter)
  - Aksi (Edit, Hapus)
- **Pagination**: Navigasi halaman untuk kategori yang banyak

### Halaman Form Kategori

Form kategori menggunakan modal/dialog untuk menambah dan mengedit data kategori.

**Elemen Form Tambah/Edit Kategori:**
- Input "Nama Kategori" (required)
- Input "Deskripsi" (textarea, optional)
- Tombol "Simpan" (primary) dan "Batal" (secondary)

### Halaman Daftar Produk

**Gambar 4.15 Rancangan Halaman Produk**

![UI Products](bab-iv/images/ui-products.png)

*Sumber: Hasil Perancangan (2026)*

**Elemen:**
- **Tombol "Tambah Produk"**: Tombol dengan ikon "+" untuk menambah produk baru
- **Filter Kategori**: Dropdown untuk memfilter produk berdasarkan kategori
- **Search Bar**: Kolom pencarian berdasarkan nama atau SKU
- **Toggle View**: Tombol untuk beralih antara tampilan grid (card) dan list (tabel)
- **Card/Grid Produk**: Setiap card menampilkan:
  - Gambar produk (thumbnail)
  - Nama produk
  - SKU (badge)
  - Harga (format Rupiah)
  - Stok (angka dengan warna: hijau jika cukup, merah jika menipis)
  - Status aktif (badge hijau/abu-abu)
  - Tombol aksi (Edit, Hapus)
- **Pagination**: Navigasi halaman

### Halaman Form Produk

Form produk menampilkan detail lengkap untuk input data produk.

**Elemen Form Tambah/Edit Produk:**
| Elemen | Tipe | Required | Validasi |
|--------|------|----------|----------|
| Nama Produk | Text Input | Ya | Maksimal 200 karakter |
| Kategori | Dropdown Select | Ya | Harus pilih kategori yang tersedia |
| SKU | Text Input | Ya | Unique, format: PRD-XXXX |
| Harga | Number Input | Ya | Minimal 0 |
| Stok | Number Input | Ya (jika tidak unlimited) | Minimal 0 |
| Unlimited Stock | Toggle Switch | Tidak | Jika aktif, stok tidak divalidasi |
| Gambar | File Upload | Tidak | Format: JPG, PNG, WEBP. Maks: 2MB |
| Deskripsi | Textarea | Tidak | - |
| Status Aktif | Toggle Switch | Tidak | Default: Aktif |
| Tombol Simpan | Button | - | Submit form |
| Tombol Batal | Button | - | Kembali ke daftar |

## 4.4.4 Transaksi POS

Halaman transaksi POS merupakan halaman terpenting dalam sistem. Halaman ini dirancang untuk memudahkan kasir dalam melayani transaksi dengan cepat dan akurat.

### Desain Halaman Transaksi POS

**Gambar 4.16 Rancangan Halaman Transaksi POS**

![UI Transactions POS](bab-iv/images/ui-transactions.png)

*Sumber: Hasil Perancangan (2026)*

### Layout Halaman POS

Halaman POS menggunakan layout dua panel (split screen) yang optimal untuk efisiensi kerja:

**Panel Kiri (70% lebar) — Daftar Produk:**
| Elemen | Deskripsi |
|--------|-----------|
| Search Bar | Pencarian real-time berdasarkan nama/SKU produk |
| Filter Kategori | Dropdown untuk memfilter produk per kategori |
| Grid Produk | Tampilan card produk dalam grid (3-4 kolom) |
| Card Produk | Gambar, nama, SKU, harga, dan tombol "Tambah" |
| Pagination | Navigasi halaman produk |
| Loading State | Animasi saat memuat produk |

**Panel Kanan (30% lebar) — Keranjang Belanja:**
| Elemen | Deskripsi |
|--------|-----------|
| Header "Keranjang" | Judul panel dengan ikon cart |
| Daftar Item | Setiap item menampilkan: nama, harga, quantity (dengan tombol + dan -), subtotal, tombol hapus |
| Total Amount | Teks besar (24px+) menampilkan total transaksi dengan format Rupiah |
| Input Customer | Text input untuk nama customer |
| Metode Pembayaran | Dropdown pilihan: Cash, Debit Card, Credit Card, E-Wallet |
| Input Pembayaran | Number input untuk jumlah yang dibayarkan |
| Kembalian | Teks otomatis yang muncul setelah input pembayaran |
| Tombol "Process Payment" | Tombol besar (full width) dengan warna hijau untuk eksekusi |
| Tombol "Clear Cart" | Tombol merah untuk mengosongkan keranjang |

### Alur Interaksi Halaman POS

1. **Mode Pencarian**: Kasir mengetik di search bar → produk terfilter real-time tanpa reload
2. **Mode Pemilihan**: Kasir mengklik card produk → produk masuk ke keranjang dengan qty=1
3. **Mode Kuantitas**: Kasir menekan tombol +/- atau input manual → subtotal berubah real-time
4. **Mode Pembayaran**: Kasir input jumlah bayar → kembalian terhitung otomatis
5. **Mode Eksekusi**: Kasir tekan "Process Payment" → konfirmasi dialog → sistem proses → struk tercetak

## Ringkasan Perancangan Antarmuka

Perancangan antarmuka Sistem POS DW menghasilkan 7 kelompok halaman utama yang mencakup seluruh kebutuhan fungsional sistem. Setiap halaman dirancang dengan memperhatikan:

1. **Konsistensi**: Seluruh halaman menggunakan komponen UI yang konsisten (tombol, form, tabel, modal) dari Livewire Flux UI
2. **Responsivitas**: Tampilan dapat menyesuaikan dengan berbagai ukuran layar
3. **User Experience**: Alur interaksi yang intuitif dengan feedback real-time
4. **Aksesibilitas**: Kontras warna yang baik, ukuran font yang terbaca, label yang jelas
5. **Efisiensi**: Shortcut dan auto-calculate untuk mempercepat kerja kasir
